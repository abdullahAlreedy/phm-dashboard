# Security and privacy

This project is a UI prototype only.

Do not place PHI in this repository, including:

- Patient names
- National IDs
- MRNs/URNs that can identify patients
- Mobile numbers
- Addresses
- Free-text clinical notes
- Raw lab rows linked to identifiable people

For GitHub Pages demos, use only:

- Synthetic patients
- Aggregated metrics
- Masked IDs that cannot be reversed
- De-identified datasets approved for demonstration

Real PHM deployment should use a secure backend, authentication, audit logs, role-based access, encrypted storage, and institutional approval.
